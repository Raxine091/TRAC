## TRAC
### IT Ticket Record Keeping System
*The very first version of a soon projected system that is to be used for tracking ticket resolution rates for the IT department*

Trac was first introduced as a system that is intended to replace the traditional record keeping using spreadsheets.

### Core functionality
- Adding ticket information
- Reading existing ticket information
- Modifying ticket details
- Deletion of ticket info
- Exporting of a .csv or .xls spreadsheet file if need be
- Printing records of filtered data as filtered
- Access control - only users authorized by the admin to use the system are given accounts and credentials which can be revoked by the admin

### Usage steps
1. An authorized users with a created user account logs in to the system
2. They land on to a dashboard which gives them an overview of the tickets that were assigned to them alongside completion rates.
3. They navigate to 'My Tickets' page - from there they can handle the CRUD of their own tickets.
4. Tickets created by them are automatically recorded as their own issues to sort
5. Only a user with admin privileges granted by an admin can view all tickets and the department workload therefore enabling them to assign roles accordingly
6. In any case a user may wish to change their credentials they may do so in the 'My Account' page.

### Environment
Due to data privacy and such related concerns, the system is designed to run over the organisations network only.  
No requests from outside the network can access the information logs of the activities.

### Flexibility and Scalability
On the first setup, the admin sets the structure of their Organization - Number of departments and company name.

### Transition
Transition into TRAC is simplified. Old records on spreadsheets may be imported to have all records in one place.
